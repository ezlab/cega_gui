CEGA Overview

CEGA, (Conserved Elements from Genomic Alignments), is a database of conserved vertebrate elements. The aim of this database is to provide acces to precomputed sets of conserved sequences from different species and at different levels of the vertebrate phylogeny. The identification of these conserved elements is based on the delineation of colinear blocks of 5 representative species (including human and one species at the root level of the clade), the multiple sequence alignment of these blocks using a global-local strategy (Mlagan) and the subsequent computation of elements using PhastCons.

Search Parameters

Search

  1. Select the clade level on which to make your query: Vertebrates, Amniotes, Eutheria, Boreoeutheria and Euarchontolires.
  2. Select your species of interest.
  3. Input a your gene or genomic region of interest.

    4. Possible queries:
    • No input displays all elements for this species, example , here for all vertebrate human noncoding elements.
    • Genomic interval such as chr2:100053307-200055635
    • Chromosome name such as chr4
    • Gene name such as hoxd1 displays elements in the gene location +/- 500bp.
    • CEGA element ID such as 2.65
    • Use the chromosome view on the home page to select a region of interest. First click on a block and then you can expand the desired region. The selection appears as genomic coordinates on the right panel. Click on submit to obtain results of conserved elements in that location.

Filters

Filters can be used to obtain more specific list of CEGA elements.
Possible filters are:

  • Length: define a range from - to
  • Score: PhastCons score, define a range from - to (See Score description below)
  • Status: Chose to display all conserved elements or only noncoding elements, only coding, only intronic or intergenic elements based on annotation.

Results Display

Table

The table displays all CEGA conserved elements for the request with filters. Elements of each row can be further investigated by clicking on the row. Multiple sequence alignment is shown in the row view as well as visualization of the element genomic surrounding region with a screenshot from UCSC. For each request, you can directly access this region with the CEGA track on UCSC or download bed or fasta for the selected elements.

Score

The score is the PhastCons score for the element. Phastcons uses a phylo-HMM with a conserved and a non-conserved state. The conserved elements from CEGA are likely to follow the conserved state of the model. The scores are log-odds scores of the probability under the conserved model minus the probability under the non-conserved model, with a higher score reflecting a higher probability of the element being under the conserved status.

Status

The status shows if the element is coding or noncoding and also if it is intergenic or intronic. Some long non-coding RNAs or miRNAs are also annotated as such. These status are based on annotation from ENSEMBL genes, GENCODE comprehensive set, Human Body Map lincRNAs, and miRBase

DNase

DNase shows DNA accessibility score from ENCODE based on the merging of the score for 9 human cell lines. For species other than human, the human data for this element is shown. The grayscale represents the score percentage, black = 100% and white = 0% or data missing.

Reference:

TFBS

The TFBS columns shows the maximum number of transcription fators that were reported as bound to a site overlapping the element from the Uniform Peaks from ENCODE.

Other DB

Other DB column shows the presence of overlapping element in other known database about conserved elements. U is for UCNEbase, V is for Vista Enhancer Database, C is for Condor database
